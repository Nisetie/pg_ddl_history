# pg_ddl_history
Журналирование DDL-команд

## Описание
Расширение предназначено для сохранения выполняемых DDL-команд в отдельной таблице. В дальнейшем информацию из этой таблицы можно использовать для анализа изменений, выяснения виновника поломки БД или для фиксации правок для их переноса между различными ландшафтами одной и той же системы.

## Основные объекты для использования

- changes.history. В эту таблицу записываются все запускаемые DDL-команды
- changes.version. В эту таблицу записываются фиксации контрольных точек в истории изменений БД. Полезна для группировки изменений.
- changes.checkpoint(). Процедура присваивает всем незафиксированным записям в changes.history инкремент версии изменений из changes.version.
